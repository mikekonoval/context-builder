# context-builder

Claude skill, который собирает папку `context/` в любом проекте — три подпапки, фиксирующие замысел, чертёж и бизнес-контекст. Эту папку агент читает перед каждой задачей вместо того, чтобы каждый чат начинать с нуля.

## Что получится в проекте

```
project-root/
├── CLAUDE.md           ← скилл добавит секцию «Project context» со ссылкой на context/
└── context/
    ├── INDEX.md        ← навигация по всему context/
    ├── idea/           ← замысел: супер-идея, аудитория, MVP, риски, принципы
    ├── architecture/   ← чертёж: пресс-релиз, карта системы, модель данных, стек, roadmap
    └── business/       ← бизнес: продукты, цены, юнит-экономика, маркетинг, бренд
```

## Зачем

Агент в проекте без `context/` каждый раз спрашивает: «а кто целевая аудитория?», «какой тон коммуникации?», «какие цели у вас на квартал?». С `context/` — читает один нужный файл и работает.

## Два режима

**Mode A (с нуля).** Папка пустая или код только начинается → скилл ведёт через интервью, последовательно собирая три фазы (idea → architecture → business). На каждой фазе — несколько стейджей с автогенерируемыми вопросами «по одному за раз, с 3-4 вариантами ответа».

**Mode B (существующий проект).** Скилл находит `src/`, `package.json` или непустой README → объявляет режим анализа, спрашивает подтверждения, читает код, пакеджи, тесты, существующий CLAUDE.md → заполняет всё что выводится (с пометкой `[авто-выведено: <источник>]`), интервью только на дыры.

Между двумя — auto-detect: скилл сам решает, какой режим включить, и подтверждает с пользователем.

## Установка

Скопируй папку `context-builder/` в одно из мест:

- **Глобально:** `~/.claude/skills/context-builder/` — доступен во всех проектах
- **Локально:** `<project>/.claude/skills/context-builder/` — только в одном проекте

После этого Claude Code увидит скилл при следующей сессии.

## Использование

Триггеры на русском или английском:

| Что хочу | Триггер |
|---|---|
| Собрать context с нуля или проанализировать существующий код | `собери context`, `настрой context для проекта`, `build project context` |
| Перезаполнить одну из фаз | `обнови idea`, `обнови architecture`, `обнови business` |
| Обновить один блок внутри фазы | `обнови context/business/economics/` (и т.п.) |
| Добавить «живую» цифру (метрика, которая подтягивается свежей) | `добавь LIVE-метрику` |
| Прогнать business/ на 4 антипаттерна | `аудит business` |

## Структура этого репозитория

Этот репозиторий — **development source** скилла, не сам скилл в установленном виде.

```
context-builder/
├── SKILL.md                          ← entry-point: frontmatter, mode detection, оркестрация
└── stages/
    ├── idea.md                       ← 5 stages для context/idea/ (4 сборка + 1 wikilinks)
    ├── architecture.md               ← 13 stages для context/architecture/ (12 сборка + 1 wikilinks)
    └── business.md                   ← 11 stages для context/business/ (8 сборка + 1 wikilinks + 2 операционных)
```

Чтобы внести правки — редактируешь файлы в `context-builder/`, потом переустанавливаешь скилл (копируешь папку заново в `~/.claude/skills/`).

## Кредиты

Методология адаптирована из курса по вайбкодингу (исходные транскрипты и материалы — приватные, не в git). Переписана в формат Claude skill: единый паттерн стейджа `Read → Create → Derive from upstream → Ask → When done`, английские структурные маркеры + русский контент в полях и вопросах.
